1. Overview

The Engram Platform brand represents a new era of AI systems that remember, learn, and think continuously. We are not just tools—we are partners in intelligence, building persistent memory and knowledge systems that scale with human ambition.

Mission

To empower developers and AI teams to build intelligent agents with persistent memory, enabling systems that learn from context, remember interactions, and make better decisions over time.

Primary Tagline

Memory that thinks with you.

Positioning

Engram is the infrastructure layer for intelligent, memory-augmented AI systems. We sit between raw vector databases and application logic, providing semantic reasoning, temporal persistence, and multi-layer knowledge orchestration.

Brand Pillars

Intelligence
We don't just store vectors—we enable reasoning across multi-layer knowledge graphs, understanding connections that matter.
Persistence
Memory that lasts. Temporal depth. Systems that remember across sessions, learning and evolving with every interaction.
Precision
Exact retrieval. No noise. Technical rigor in every layer—from embeddings to graph topology to API design.
Trust
Transparent memory. Auditable reasoning. Systems that work because developers understand exactly how they remember.

6. Motion & Animation

Motion in Engram is purposeful, not decorative. Animations reveal relationships, guide attention, and create a sense of living, responsive systems. All animations use easing and timing that feel technical yet approachable.

Animation Principles

  • Duration: 300–500ms for UI interactions, 1–2s for data-driven animations
  • Easing: cubic-bezier(0.4, 0, 0.2, 1) for standard transitions, ease-out for entrances
  • Purpose: Each animation answers a user question or reveals hidden relationships
  • Restraint: Subtle > flashy. Never distract from content

Six Core Animations

1. Node Pulse

Indicates an active memory node or data point. Used in dashboards to show "live" systems.

@keyframes nodePulse { 0%, 100% { r: 8; opacity: 1; } 50% { r: 16; opacity: 0.4; } } animation: nodePulse 2s ease-in-out infinite;

2. Edge Draw (Path Animation)

Animates the drawing of knowledge graph connections. Reveals relationships progressively.

@keyframes edgeDraw { 0% { stroke-dashoffset: 100; } 100% { stroke-dashoffset: 0; } } stroke-dasharray: 100; animation: edgeDraw 1.5s ease-out forwards;

3. Data Stream (Particle Float)

Particles rising through data. Visualizes information flow and processing.

@keyframes dataFloat { 0% { transform: translateY(0); opacity: 1; } 100% { transform: translateY(-180px); opacity: 0; } } animation: dataFloat 2s ease-out forwards;

4. Vector Cluster Morph

Nodes reorganizing into clusters. Shows semantic grouping and relationships.

@keyframes clusterMove { 0% { cx: var(--start-x); cy: var(--start-y); } 100% { cx: var(--end-x); cy: var(--end-y); } } animation: clusterMove 1.5s ease-in-out infinite alternate;

5. Memory Consolidation (Nodes Merging)

Two nodes merging into one. Represents memory consolidation and synthesis.

@keyframes mergeLeft { 0% { cx: 40; opacity: 1; } 100% { cx: 100; r: 0; opacity: 0; } }

6. Retrieval Ping (Ripple)

Concentric circles emanating from a source. Shows query propagation and retrieval.

@keyframes ripple { 0% { r: 0; opacity: 1; } 100% { r: 60; opacity: 0; } } animation: ripple 1.5s ease-out infinite;

7. Voice & Tone

Engram's voice is technical yet approachable, confident yet humble. We speak to developers and AI engineers with respect for their expertise while remaining accessible to those newer to the field.

Voice Pillars

Technical Precision

We use exact language. Vector, embedding, retrieval augmented generation—not "AI magic" or "smart memory."

Confident but Not Arrogant

We know our platform. We don't overclaim. We say what we do well and acknowledge trade-offs.

Helpful, Not Patronizing

Assume the reader is smart. Explain complexity clearly, not by dumbing it down.

Direct and Action-Oriented

No fluff. Start with the what, then explain the why. Use imperatives: "Initialize," "Configure," "Deploy."

Vocabulary: Do's & Don'ts

✓ Use
  • Vector embedding, semantic search
  • Multi-layer knowledge graph
  • Retrieval-augmented generation (RAG)
  • Temporal persistence, memory consolidation
  • API, SDK, integration
  • Nodes, edges, topology
  • Inference, reasoning, inference engine
✗ Avoid
  • AI magic, magic memory, smart AI
  • Brain metaphors (neural, synapse)
  • Buzzwords (paradigm shift, synergy)
  • Vague language (remembers stuff, learns)
  • Tool, gadget, platform (overused)
  • Network, web, mesh (ambiguous)
  • Black box, mysterious, unexplainable

Layout Principles

  • Margin collapse awareness: Use padding on containers, margins on children to avoid unexpected collapses
  • White space is content: Never fill silence with decoration. Breathing room clarifies hierarchy
  • Max-width constraints: Content max-width 1200px for reading, 1400px for data-dense layouts

  • Horizontal breathing: Minimum 16px padding on mobile, 24px on tablet, 32px on desktop

11. Dashboard UI Guide

Engram dashboards prioritize data density and clarity. Dark backgrounds reduce eye strain during extended monitoring sessions. Visual hierarchy guides attention to critical metrics.

Dashboard Design Principles

Dark Mode First
  • • All dashboards assume dark background
  • • Minimize brightness variation (contrast, not saturation)
  • • Use semantic colors sparingly
  • • Status indicators should be glanceable
Information Density
  • • Show 15–20 key metrics per view
  • • Group related metrics into sections
  • • Hide secondary data behind expand/detail
  • • Use sparklines for trends, not large charts
Status Clarity
  • • Green = healthy, running, active
  • • Amber = attention needed, degraded
  • • Red = critical, down, error
  • • Gray = disabled, inactive, N/A
Panel Hierarchy
  • • Hero panels: 2x2 grid (top left)
  • • Secondary panels: 3-column grid
  • • Detail panels: Full-width tables
  • • Borders emphasize importance

15. Illustration Style

Illustrations support conceptual understanding without overexplaining. Technical diagrams use consistent line weights and a limited palette. Isometric views emphasize three-dimensionality.

Illustration Principles

Technical Precision
  • • Consistent stroke weight (1.5–2px)
  • • Geometric base shapes
  • • Alignment to grid (8px minimum)
  • • No decorative flourishes
Colour Usage
  • • Primary: Rust (#C25216)
  • • Secondary: Amber (#E8A030)
  • • Accent: Success/Warning (semantic)
  • • Lines: Graphite (#8892A4)

Illustration Types

Isometric Diagrams

Show spatial relationships and data flow. Use 30-degree angles. Perfect for platform architecture, embedding spaces, and multi-layer memory representations.

Node Graphs

Knowledge graphs, semantic networks, connections. Nodes: circles or squares. Edges: straight lines or curves. Emphasize topology, not aesthetics.

Vector Fields

Direction and magnitude of embeddings. Use arrows or flow lines. Suggest dimensionality without overcomplicating. Works well for abstract "memory space" visuals.

Timeline Diagrams

Temporal progression, event sequences. Horizontal or vertical axes. Nodes for milestones, lines for progression. Show causality or dependency.

Line Weight & Scale

Element Line Weight Context
Primary strokes (nodes, shapes) 1.5–2px Main visual elements
Edges / connections 1–1.5px Links between nodes
Detail / shadow strokes 0.75–1px Subtle emphasis, depth
Text indicators / labels N/A Use font weight for emphasis

19. Accessibility

Engram brand must be accessible to all users. WCAG 2.2 AA compliance is baseline. All colour combinations, type sizes, and interactive elements must meet accessibility standards.

Ensure all animated content can be disabled. Provide skip links for navigation. Test with screen readers (NVDA, JAWS, VoiceOver).

Accessibility Checklist

  • ✓ All images have descriptive alt text
  • ✓ Colour is not the only differentiator (use icons, text, patterns)
  • ✓ Form labels are explicitly associated with inputs
  • ✓ Keyboard navigation works (Tab order logical)
  • ✓ Focus indicators are visible and clear
  • ✓ Minimum touch target size: 48×48px
  • ✓ All interactive elements have :focus-visible states
  • ✓ Animations can be disabled (prefers-reduced-motion)
  • ✓ Text resizing works to 200%
  • ✓ No automatic audio playback or redirects without warning
